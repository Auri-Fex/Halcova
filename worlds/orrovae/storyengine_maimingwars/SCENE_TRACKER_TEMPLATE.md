## 2. CONTENT RATING SYSTEM

### Rating Scale (0-5)

| Rating | Intensity | Description |
|--------|-----------|-------------|
| 0 | None | Element not present |
| 1 | Mild | Implied or briefly mentioned |
| 2 | Moderate | Present but not detailed |
| 3 | Significant | Detailed, plot-relevant |
| 4 | Intense | Graphic, extended focus |
| 5 | Extreme | Unflinching, potentially disturbing |

### Content Categories &amp; Tags

#### VIOLENCE [V]
```
V-COMBAT       Armed/unarmed fighting
V-TORTURE      Deliberate infliction of pain
V-EXECUTION    Premeditated killing
V-MASSACRE     Mass violence
V-MAIMING      Permanent disfigurement (central to Maiming Wars)
V-SELFHARM     Self-inflicted violence
V-CHILD        Violence involving minors
V-ANIMAL       Violence to animals/creatures
```

#### GORE [G]
```
G-BLOOD        Blood and bleeding
G-WOUND        Visible injuries described
G-ORGAN        Internal organs visible/described
G-DISMEMBER    Limb loss or separation
G-DECAY        Decomposition, rot
G-MEDICAL      Surgical/medical gore
G-SÄIDE        Symbiote-specific body horror (seamwork, drift, shedding)
```

#### HORROR [H]
```
H-BODY         Body horror (transformation, corruption)
H-PSYCHO       Psychological horror
H-CREATURE     Monster/creature horror
H-COSMIC       Existential/cosmic dread
H-HOLLOW       Hollowing-specific horror
H-SURFACE      Surface-creature encounters
H-CLAUSTRO     Claustrophobic horror (underground specific)
H-UNCANNY      Uncanny valley (constructs, partially Hollowed)
```

#### SEXUAL [S]
```
S-TENSION      Sexual tension, no action
S-IMPLY        Implied sexual activity (fade to black)
S-PARTIAL      Some explicit detail
S-EXPLICIT     Fully explicit scene
S-NONCON       Non-consensual elements (assault, coercion)
S-TRANSACT     Transactional sex (prostitution, political)
S-KINK         BDSM or kink elements
S-SÄIDE        Symbiote-related intimacy (resonance, shared sensation)
```

#### PSYCHOLOGICAL [P]
```
P-TRAUMA       Trauma response, PTSD
P-GRIEF        Intense grief, loss
P-ABUSE        Psychological abuse, manipulation
P-IDENTITY     Identity crisis, dissociation
P-HOLLOW       Hollowing-related psychological horror
P-ADDICTION    Addiction, dependency
P-ISOLATION    Extreme isolation, loneliness
P-DESPAIR      Hopelessness, suicidal ideation
```

#### DISTURBING [D]
```
D-MORAL        Moral injury, being forced to do terrible things
D-COMPLICIT    Complicity in atrocity
D-CHILD        Content involving harm to children
D-BETRAYAL     Devastating betrayal
D-FUTILITY     Overwhelming futility
D-SYSTEM       Systemic horror (bureaucratic evil)
D-CONSENT      Violations of bodily autonomy (forced Hollowing, maiming)
```

### Scene Rating Example

```
Violence:       4 [V-COMBAT, V-MAIMING, V-TORTURE]
Gore:           4 [G-WOUND, G-DISMEMBER, G-SÄIDE]
Horror:         3 [H-BODY, H-HOLLOW]
Sexual:         0
Psychological:  3 [P-TRAUMA, P-IDENTITY]
Disturbing:     4 [D-CONSENT, D-MORAL]
───────────────
Content Notes:  POV character forced to maim ally to prevent Hollowing. 
                Extended focus on Säide attempting repair and failing.
                Dark humor in scene: gallows joke about "at least it's your off-hand."
```

## 3. FORESHADOWING TRACKING

### Seed Types

| Code | Type | Description | Example |
|------|------|-------------|---------|
| OBJ | Object | Physical item introduced | Surface artifact that will become weapon |
| INFO | Information | Fact, secret, rumor | Mention of "gate protocols" |
| REL | Relationship | Connection established | Kaisa meets surface creature without violence |
| SKILL | Ability | Skill or trait shown | Torven's ability to read Säide-script |
| PAT | Pattern | Recurring element | Corpse arrangements by surface creatures |
| THEME | Thematic | Idea seeded | "What the Säide can't fix..." |
| CHAR | Character | Person introduced/mentioned | Naming someone who appears later |
| LOC | Location | Place mentioned/visited | The "old gate" Kaisa remembers |

### Visibility Scale

| Code | Visibility | Reader Should... |
|------|------------|------------------|
| HIDDEN | Very Subtle | Notice only on reread |
| SUBTLE | Low-key | Possibly notice, not remember |
| VISIBLE | Noticeable | Remember but not guess payoff |
| OBVIOUS | Pointed | Expect payoff, guess wrong about nature |
| EXPLICIT | Direct | Clearly setup for later |

### Seed Registry Format

Each foreshadowing element gets a unique ID for tracking:

```
SEED ID:        [B#-S###] (e.g., B1-S042 = Book 1, Seed #42)
Seed Name:      [Descriptive name]
Planted:        [Scene ID where planted]
Type:           [OBJ/INFO/REL/SKILL/PAT/THEME/CHAR/LOC]
Visibility:     [HIDDEN/SUBTLE/VISIBLE/OBVIOUS/EXPLICIT]
Element:        [What is the seed?]
Target Payoff:  [Scene ID where it pays off]
Payoff Type:    [Revelation/Reversal/Callback/Resolution/Irony]
True/False:     [True clue / Red herring / Misdirection]
Notes:          [Additional context]
```

### Payoff Execution Format

```
PAYOFF ID:      [B#-P###]
Seed ID:        [Reference to original seed]
Executed:       [Scene ID]
Original Plant: [Brief description]
Execution:      [How the payoff occurs]
Satisfaction:   [Setup-worthy / Earned / Surprising / Subverted]
Reread Value:   [How this enhances rereading]
```

## 4. RELATIONSHIP TRACKING

### Relationship Registry

Each significant relationship gets tracked across the series:

```
RELATIONSHIP ID:    [R###]
Characters:         [Character A] ↔ [Character B]
Type:               [Romantic/Familial/Professional/Adversarial/Complicated]
Lane Intersection:  [Which POV lanes does this cross?]
Start State:        [Relationship at first appearance]
End State:          [Relationship at series end or death]
Key Scenes:         [Scene IDs of major beats]
```

### Beat Types

| Code | Beat Type | Description |
|------|-----------|-------------|
| MEET | First Meeting | Characters meet for first time |
| BUILD | Building | Relationship strengthening |
| STRAIN | Straining | Tension or conflict introduced |
| BREAK | Breaking | Major rupture or betrayal |
| REPAIR | Repairing | Reconciliation or healing |
| DEEPEN | Deepening | Moving to new level of intimacy/trust |
| TEST | Testing | Relationship tested by external pressure |
| REVEAL | Revelation | Secret revealed that changes relationship |
| LOSS | Loss | Relationship ended by death/separation |
| TRANSFORM | Transformation | Fundamental change in relationship nature |

### Scene Beat Format

```
Relationship:   [R### - Names]
Beat Type:      [Code + description]
Before State:   [Relationship status entering scene]
Scene Event:    [What happens]
After State:    [Relationship status exiting scene]
Shift:          [+3 to -3 scale: major positive to major negative]
Future Setup:   [What this enables later]
```

### Subplot Types

| Type | Function | Example |
|------|----------|---------|
| MIRROR | Reflects main theme from different angle | Construct awakening mirrors human identity crisis |
| CONTRAST | Shows alternative approach/outcome | Eneko's pacifism vs. Torven's violence |
| COMPLICATE | Creates obstacles or competing demands | Sigurd's research vs. Sigurd's patients |
| DEEPEN | Reveals character through different context | Kaisa's family history through flashbacks |
| CONNECT | Links POV lanes or installments | Shared antagonist across multiple lanes |

## 7. WORLDBUILDING DEPENDENCIES

### Dependency Types

| Code | Type | Description |
|------|------|-------------|
| LOC | Location | Specific place needs development |
| TECH | Technology | Technology/infrastructure detail needed |
| SÄIDE | Säidesolu | Symbiote mechanics, strains, etc. |
| CULT | Culture | Cultural practice, doctrine, tradition |
| HIST | History | Historical event or period |
| CHAR | Character | Background character needs development |
| LANG | Language | Conlang terms, naming, dialogue |
| BIO | Biology | Creature, plant, biological system |
| POL | Political | Faction, institution, power structure |
| ECON | Economic | Trade, resource, economic system |

### Dependency Status

| Status | Meaning |
|--------|---------|
| NEEDED | Not yet developed, scene requires it |
| PARTIAL | Some development exists, needs expansion |
| EXISTS | Fully developed in canon |
| CONFLICT | Existing canon conflicts with scene needs |
| FLEXIBLE | Can be developed to fit scene |

### Scene Dependency Format

```

Status:         [Status]
Canon Ref:      [If exists: file/section reference]
Scene Need:     [What the scene requires]

```

## USAGE WORKFLOW

1. **Planning Phase**: Create book overview, map major beats
2. **Scene Breakdown**: Create scene cards for each chapter
3. **Dependency Audit**: Flag worldbuilding needs
4. **Registry Population**: Add seeds, relationships, subplots
5. **Cross-Reference**: Build indices as scenes accumulate
6. **Continuity Check**: Verify scene-to-scene consistency
7. **Content Audit**: Review content distribution
8. **Revision Tracking**: Update cards through drafts
